    - applyProposals must re-sort by order and re-number sequentially (0, 1, 2...) after all mutations
  </behavior>

Create `components/card-news/structure/StructureSystem.tsx` mirroring the structure of `components/card-news/quality/EvaluationSystem.tsx` exactly (same section headers, same hook pattern, 'use client' directive, useCallback for all actions).

File structure:
1. `'use client'` directive
2. Imports: `useState`, `useCallback` from react; `CardNewsItem` from `@/components/lib/types`

   - Export `ProposalType = 'reorder' | 'add' | 'delete' | 'edit'`
   - Export `StructureProposal` interface with all fields (id, type, source, reason, fromOrder?, toOrder?, insertAfterOrder?, newCard? as `Omit<CardNewsItem, 'order'>`, targetOrder?, newHeadline?, newSubtext?)
   - Export `StructureState` interface: phase ('idle'|'reviewing'|'proposed'|'applying'|'done'|'error'), proposals: StructureProposal[], acceptedIds: string[] (array form of Set for useState compatibility), error: string | null, isReviewing: boolean
4. Section: Utilities
   - Export `applyProposals(cards: CardNewsItem[], acceptedProposals: StructureProposal[]): CardNewsItem[]`
     - Step 1: build a Set of accepted delete targetOrders for conflict detection
     - Step 2: Apply 'edit' proposals — for each accepted edit, if its targetOrder is NOT also targeted by an accepted delete, update headline/subtext on matching card
     - Step 3: Apply 'delete' proposals — filter out cards whose order matches any accepted delete targetOrder
     - Step 4: Re-normalize order after delete (map((c, i) => ({...c, order: i})))
     - Step 5: Apply 'add' proposals — for each accepted add, insert newCard after card at insertAfterOrder; if no match, append at end
     - Step 6: Re-normalize order after add
     - Step 7: Apply 'reorder' proposals — for each accepted reorder, find card with fromOrder and set its order to toOrder, and card with toOrder to fromOrder (swap)
     - Step 8: Final re-sort and re-normalize: result.sort((a,b)=>a.order-b.order).map((c,i)=>({...c,order:i}))
5. Section: Initial State Factory
   - Export `createInitialStructureState(): StructureState`
6. Section: Hook
   - Export `useStructureSystem()` using useState + useCallback
   - Actions: startReview (phase→'reviewing', isReviewing→true, error→null), setProposals (phase→'proposed', isReviewing→false, acceptedIds initialized to ALL proposal IDs), toggleAccept, acceptAll, rejectAll, applyAccepted (phase→'applying' then call applyProposals + phase→'done'), setError, reset

3. In POST handler:
   - Parse request body: { topic, audience, apiKey }
   - Validate required fields (topic, audience must not be empty)
   - Override the client apiKey with the one from request body (for client-side API key input)
   - Configure web_search tool:
     { type: 'web_search', display_name: 'web_search' }

4. Create first API call for research:
   - System prompt: "You are a research assistant. Search for at least 3 credible sources with recent statistics, examples, or case studies about the topic. Include specific URLs for citation."
   - User prompt: "Research the following topic for card news content: {topic}. Target Audience: {audience}. Search for at least 3 credible sources with recent statistics, examples, or case studies."
   - Include tools array with web_search tool
   - Include anthropic-beta: tools-2025-09-04 header

5. Parse research results from response.content[0].text

6. Create second API call for copy generation:
   - System prompt: "You are a professional copywriter specializing in Instagram card news. Generate card news in the following structure: 1 cover card (hook), 4-6 body cards (each with 1 headline + 2-3 sentences), 1 CTA card. Each card: headline 1 line, subtext 2-3 lines."
   - User prompt: "Based on the following research, generate card news copy. Topic: {topic}. Target Audience: {audience}. Research Context: {researchContext}. Format as JSON with 'cards' array (type, headline, subtext, order) and 'researchSources' array (title, url, summary)."

7. Parse JSON response using JSON.parse()

8. Return NextResponse.json with parsed data

9. Handle errors: return 500 status with error message

Do NOT use NEXT_PUBLIC_ prefix for API key - this must be server-side only.</action>

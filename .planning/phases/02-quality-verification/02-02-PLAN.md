그 다음, `cards.length > 0` 조건 블록 내부에서 `CardNewsList` 다음에 `QualityLoop`를 렌더링한다.

현재 구조:
```tsx
{cards.length > 0 && (
  <>
    <hr className="border-border" />
    <CardNewsList cards={cards} onUpdate={handleUpdateCards} />
  </>
)}
```

수정 후 구조:
```tsx
{cards.length > 0 && (
  <>
    <hr className="border-border" />
    <CardNewsList cards={cards} onUpdate={handleUpdateCards} />
    <hr className="border-border" />
    <QualityLoop
      cards={cards}
      apiKey={apiKey}
      onApproved={(approvedCards) => setCards(approvedCards)}
      onMaxLoopsReached={(finalCards) => setCards(finalCards)}
    />
  </>
)}
```

주의: `handleUpdateCards` 기존 함수는 그대로 유지한다. QualityLoop의 onApproved와 onMaxLoopsReached는 인라인 arrow function으로 처리하여 승인된 카드를 cards 상태에 반영한다. 다른 import, state, 핸들러는 변경하지 않는다.
